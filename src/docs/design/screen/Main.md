# 메인 화면 설계서

**화면 설명**: 서비스의 메인 페이지로, 현재 진행 중인 토의실 목록을 보여주고 새로운 토의실을 생성할 수 있는 진입점을 제공합니다.

**진입 경로**: 로그인 후 첫 화면

---

## 화면 요소

| 요소 | 타입 | 설명 | 구현 상세 |
| --- | --- | --- | --- |
| 헤더 | Header | 상단 네비게이션 바 | 서비스 로고, 탭(전체/참여), 마이룸 링크, 로그인/회원가입 버튼 |
| 히어로 섹션 | Section | 서비스 요약 및 방 생성 유도 | 시각적 그라데이션 카드, "새로운 토의방 만들기" 버튼 포함 |
| 방 생성 모달 | Modal | 신규 토의방 정보 입력창 | 제목, 지역, 설명, 접근 권한(모두/공무원) 설정 |
| 토의실 목록 탭 | Tab | 목록 필터링 | '전체논의방', '내가참여중' 두 가지 상태 지원 |
| 정렬 드롭다운 | Dropdown | 목록 정렬 순서 변경 | '최신순' 등 (UI 위치만 구현됨) |
| 토의실 카드 | Card | 개별 토의방 요약 정보 | 지역, 공식 뱃지(공무원 생성 시), 제목, 내용 요약, 참여자 수, 기간 |
| 페이지네이션 | Pagination | 목록 페이지 이동 | 활성 페이지 강조, 이전/다음 버튼 포함 |
| 푸터 | Footer | 하단 정보 영역 | 저작권 정보, 서비스 정보 링크 |

---

## 사용자 동작 및 인터랙션

| 동작 | 결과 | 예외 처리 및 피드백 |
| --- | --- | --- |
| 탭 클릭 | '전체' 또는 '참여 중'으로 목록 필터링 | 각 탭 상태에 맞춰 `Header`와 `RoomListSection` 업데이트 |
| 방 만들기 클릭 | 방 생성 모달 오픈 | - |
| 토의실 카드 클릭 | 해당 채팅방 페이지로 이동 | - |
| 마이룸 링크 클릭 | 마이룸 페이지('/myroom')로 이동 | - |

## 디자인 시스템 적용
- **레이아웃**: `max-w-[1280px]` 중앙 정렬 구조
- **히어로 카드**: `hero-gradient` 및 부유하는 원형 장식 효과 적용
- **그리드**: 반응형 그리드 시스템 사용 (모바일 1컬럼, 태블릿 2컬럼, 데스크탑 3컬럼)
