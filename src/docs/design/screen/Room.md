# 토의방(채팅 및 제안서) 화면 설계서

**화면 설명**: 참여자들과 실시간으로 대화하고, 제안서를 작성하거나 열람할 수 있는 통합 워크스페이스 화면입니다.

**진입 경로**: 메인 화면 → 토의실 카드 클릭

---

## 화면 요소

| 요소 | 타입 | 설명 | 구현 상세 |
| --- | --- | --- | --- |
| 방 정보 헤더 | Header | 방 제목 및 상태 정보 | 제목, 지역, 참여 인원 수 표시 |
| 채팅 영역 | Chat | 실시간 메시지 목록 | 보낸 사람별(방장/공무원/일반) 프로필 스타일 및 메시지 버블 |
| 채팅 입력창 | Form/Textarea | 메시지 작성 및 전송 | `Shift + Enter` 줄바꿈 지원, 전송 버튼 포함 |
| 메뉴 버튼 | Button | 사이드 메뉴 열기/닫기 | 햄버거 아이콘, 참여자 목록 및 관리 기능 포함 |
| 사이드 메뉴 | Drawer | 상세 정보 및 버튼 모음 | 참여자 목록, 제안서 열기, 방 나가기, (관리자용) 강퇴/삭제 |
| 제안서 패널 | Aside | 제안서 작성/열람용 슬라이드 | 오른쪽 35% 영역 차지, 제목/문제/솔루션/첨부파일 입력 |
| 제안서 관리 버튼 | Button | 임시저장 및 최종 제출 | 패널 하단 고정, 제출 시 확인 모달 오픈 |

---

## 사용자 동작 및 인터랙션

| 동작 | 결과 | 예외 처리 및 피드백 |
| --- | --- | --- |
| 메시지 전송 | 채팅 목록에 즉시 추가 | 입력 내용 없을 시 전송 방지 |
| 제안서 열기(메뉴) | 제안서 패널 슬라이드 인 | 이미 열려있는 경우 상태 유지 |
| 최종 제출 클릭 | 제출 확인 모달 오픈 | "제출 후 수정 불가" 안내 포함 |
| 방 나가기 클릭 | 퇴장 확인 모달 오픈 | 확인 시 메인 화면으로 이동 |
| 강퇴/삭제(관리자) | 각 기능별 전용 확인 모달 오픈 | 삭제 시 모든 데이터 파기 안내 |

---

## 디자인 시스템 적용
- **레이아웃**: `h-screen`, `overflow-hidden`으로 전체 화면 고정 구조
- **채팅 배경**: `#FAFAFA` (연회색) 계열로 가독성 확보
- **역할별 색상**: 방장(보라), 공무원(파랑), 일반(회색) 계열 프로필 배경색 차별화
- **슬라이드 효과**: `transition-transform duration-300`으로 부드러운 패널 전환
