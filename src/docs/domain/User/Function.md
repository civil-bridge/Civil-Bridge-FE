# User Domain Function Specification

이 문서는 사용자 인증 및 프로필 관리 관련 기능 요구사항을 정의합니다.

---

## F-USR-01: 로그인
**설명**: 가입한 계정으로 서비스에 접속합니다.
**상세 요구사항**:
- 아이디(loginId)와 비밀번호를 입력받습니다.
- 로그인 성공 시 Access Token과 Refresh Token을 발급받아 저장합니다.
- 로그인 성공 후 [F-ROM-01] 메인화면으로 이동합니다.
- 실패 시(계정 불일치 등) 적절한 에러 메시지를 표시합니다.

---

## F-USR-02: 회원가입
**설명**: 새로운 계정을 생성합니다.
**상세 요구사항**:
- 필수 입력값: 아이디, 비밀번호, 이름, 닉네임, 전화번호, 이메일
- 이메일 중복 확인 및 인증번호 발송/검증 프로세스가 포함됩니다.
- 비밀번호와 비밀번호 확인 필드가 일치해야 합니다.
- 가입 완료 후 로그인 화면으로 이동합니다.

---

## F-USR-03: 내 정보 관리 (프로필)
**설명**: 사용자의 기본 정보를 조회하고 관리합니다.
**상세 요구사항**:
- 닉네임, 이메일, 공무원 인증 상태를 표시합니다.
- 프로필 수정: 닉네임 변경, 프로필 이미지 변경 등이 가능합니다.
- 로그아웃: 저장된 토큰을 삭제하고 로그인 화면으로 이동합니다.

---

## F-USR-04: 공무원 인증
**설명**: @go.kr 등의 기관 메일을 통해 공무원 신분을 인증합니다.
**상세 요구사항**:
- 공무원 이메일을 입력받아 인증 코드를 발송합니다.
- 발송된 코드를 올바르게 입력하면 사용자 그룹이 'OFFICIAL'로 변경됩니다.
- 인증 성공 시 [F-ROM-01] 등의 화면에서 공무원 전용 태그가 활성화됩니다.
