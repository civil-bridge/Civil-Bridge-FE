# Proposal Domain Function Specification

이 문서는 시민 제안서 작성 및 동의 투표 관련 기능 요구사항을 정의합니다.

---

## F-PPS-01: 제안서 초안 작성 및 임시 저장
**설명**: 논의방의 대화 내용을 바탕으로 구체적인 해결 방안을 담은 제안서를 작성합니다.
**상세 요구사항**:
- 작성 권한: 논의방의 `LEADER`만 편집 가능하지만, 다른 참여자들은 작성 과정을 실시간으로 볼 수 있습니다.
- 입력 항목: 제안서 제목, 발생 중인 문제(Issue), 해결 방안(Solution), 예상 효과(Expected Effect), 참고 자료(Attachments).
- `PATCH /api/...` 엔드포인트를 통해 실시간 혹은 수동으로 초안을 서버에 저장할 수 있습니다.

---

## F-PPS-02: 제안서 최종 제출 및 투표 설정
**설명**: 작성이 완료된 제안서를 투표 단계로 전환합니다.
**상세 요구사항**:
- 제출 시 `최소 동의 인원수`와 `투표 마감 기한`을 설정해야 합니다.
- 제출이 완료되면 제안서의 상태는 `VOTING`으로 변경되며, 더 이상 내용을 수정할 수 없습니다.

---

## F-PPS-03: 제안서 동의 투표
**설명**: 제출된 제안서에 대해 방 참여자들이 동의 의사를 표시합니다.
**상세 요구사항**:
- 현재 동의 현황(N명 동의 / M명 목표)을 프로그레스 바 형태로 시각화합니다.
- 마감 기한까지 남은 시간을 카운트다운으로 표시합니다.
- 사용자는 한 제안서에 대해 1회만 동의 버튼을 클릭할 수 있습니다.
- 목표 인원 도달 시 '가결' 상태로 변경되며 공식 민원 사이트 링크 등이 제공될 수 있습니다.
