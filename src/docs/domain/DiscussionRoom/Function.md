# Discussion Room Domain Function Specification

이 문서는 논의방 생성 및 관리 관련 기능 요구사항을 정의합니다.

---

## F-ROM-01: 전체 논의방 목록 조회
**설명**: 시스템에 개설된 모든 공개 논의방 리스트를 조회합니다.
**상세 요구사항**:
- 최신순/인기순 등의 정렬 기능을 제공합니다.
- 페이지네이션(기본 15개)을 지원합니다.
- 각 방의 제목, 한 줄 설명, 지역(시/군/구), 현재 인원, 생성일자를 표시합니다.
- 공무원 전용 방일 경우 락(Lock) 아이콘 및 전용 배지를 표시합니다.

---

## F-ROM-02: 내가 참여 중인 논의방 목록 조회
**설명**: 로그인한 사용자가 현재 가입/참여 중인 논의방 리스트만 필터링하여 조회합니다.
**상세 요구사항**:
- [F-ROM-01]과 동일한 레이아웃을 사용하되 사용자의 가입 여부로 필터링합니다.

---

## F-ROM-03: 논의방 개설
**설명**: 새로운 토의 주제를 가진 방을 만듭니다.
**상세 요구사항**:
- 입력 항목: 방 제목(최대 50자), 상세 설명(최대 200자), 지역(2단계), 접근 권한(모두/공무원만).
- 방을 만든 사용자는 자동으로 해당 방의 `LEADER` 권한을 가집니다.

---

## F-ROM-04: 논의방 입장 및 실시간 채팅
**설명**: 특정 논의방 내부에서 참여자들과 소통합니다.
**상세 요구사항**:
- 입장 시 해당 방의 최근 대화 내역(상세 요건 미정)을 불러옵니다.
- 실시간 메시지 송수신이 가능해야 합니다.
- 참여자 목록을 확인할 수 있으며, 공무원은 특수 심볼로 구분됩니다.
- `LEADER`는 부적절한 참여자를 강퇴할 수 있는 기능을 가집니다.
- 언제든 방을 나갈 수 있으며, 마지막 한 명(LEADER)이 나갈 경우 방은 자동 삭제됩니다.
