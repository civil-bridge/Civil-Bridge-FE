# 회원가입 화면 설계서

## 1. 페이지 개요

- **URL**: `/signup`
- **목적**: 신규 사용자 계정 생성

---

## 2. 페이지 구성

### 2.1. 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                 │
│              ┌─────────────────┐                │
│              │                 │                │
│              │   회원가입 폼     │                │
│              │                 │                │
│              └─────────────────┘                │
│                                                 │
│          이미 계정이 있으신가요? 로그인            │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 중앙 정렬 레이아웃
- 최대 너비: 400px

### 2.2. UI 요소

| UI 요소 | 설명 |
|---------|------|
| 페이지 타이틀 | "회원가입" |
| 아이디 입력 | 로그인에 사용할 아이디 + 중복확인 버튼 |
| 비밀번호 입력 | 비밀번호 (마스킹 처리) |
| 비밀번호 재입력 | 비밀번호 확인 (마스킹 처리) |
| 이메일 입력 | 이메일 주소 + 인증번호 발송 버튼 (발송 시 이메일 중복검사 우선 진행) |
| 인증번호 입력 | 이메일로 받은 인증번호 + 확인 버튼 |
| 닉네임 입력 | 서비스에서 표시될 이름 + 중복확인 버튼 |
| 회원가입 버튼 | 폼 제출 |
| 로그인 링크 | 로그인 페이지로 이동 |

---

## 3. 입력 필드 상세

| 필드 | 타입 | 필수 | 플레이스홀더 | 유효성 검사 |
|------|------|------|--------------|-------------|
| 아이디 | text | ✓ | 아이디를 입력하세요 | 6자 이상, 중복 불가 |
| 비밀번호 | password | ✓ | 비밀번호를 입력하세요 | 대문자+소문자+특수문자 포함, 12자 이상 |
| 비밀번호 재입력 | password | ✓ | 비밀번호를 다시 입력하세요 | 비밀번호와 일치 |
| 이메일 | email | ✓ | 이메일을 입력하세요 | 이메일 형식, 중복 불가 |
| 인증번호 | text | ✓ | 인증번호를 입력하세요 | 발송된 인증번호와 일치 |
| 닉네임 | text | ✓ | 닉네임을 입력하세요 | 2자 이상, 중복 불가 |

## 4. 이메일 인증 흐름

### 4.1. UI 상태 변화

**초기 상태**
```
┌─────────────────────────────┬──────────────┐
│ 이메일을 입력하세요           │ 인증번호 발송 │
└─────────────────────────────┴──────────────┘
```

**인증번호 발송 클릭 시**
1. 이메일 형식 검사
2. 이메일 중복 검사 (API 호출)
3. 중복 시 → 에러 메시지 표시, 발송 안 함
4. 중복 아니면 → 인증번호 발송

**인증번호 발송 후**
```
┌─────────────────────────────┬──────────────┐
│ user@example.com (비활성화)  │ 재발송       │
└─────────────────────────────┴──────────────┘
┌─────────────────────────────┬──────────────┐
│ 인증번호를 입력하세요         │ 확인         │
└─────────────────────────────┴──────────────┘
  인증번호가 발송되었습니다. (3:00)
```

**인증 완료**
```
┌─────────────────────────────┬──────────────┐
│ user@example.com (비활성화)  │ ✓ 인증완료   │
└─────────────────────────────┴──────────────┘
```

### 4.2. 인증 흐름

1. 이메일 입력
2. "인증번호 발송" 버튼 클릭
3. 이메일 중복 검사 (중복 시 에러 메시지, 진행 중단)
4. 중복 아니면 이메일로 인증번호 발송
5. 이메일 필드 비활성화, 인증번호 입력 필드 활성화
6. 타이머 시작 (3분)
7. 인증번호 입력 후 "확인" 버튼 클릭
8. 인증 성공 시 "✓ 인증완료" 표시
9. 인증 실패 시 에러 메시지 표시
10. 타이머 만료 시 재발송 필요

## 5. 아이디 중복확인 흐름

### 5.1. UI 상태 변화

**초기 상태**
```
┌─────────────────────────────┬──────────────┐
│ 아이디를 입력하세요           │ 중복확인     │
└─────────────────────────────┴──────────────┘
```

**중복확인 완료 (사용 가능)**
```
┌─────────────────────────────┬──────────────┐
│ myusername                  │ ✓ 사용가능   │
└─────────────────────────────┴──────────────┘
  사용 가능한 아이디입니다.
```

**중복확인 완료 (사용 불가)**
```
┌─────────────────────────────┬──────────────┐
│ myusername                  │ 중복확인     │
└─────────────────────────────┴──────────────┘
  이미 사용 중인 아이디입니다.
```

## 7. 사용자 상호작용

| 상호작용 | 결과 |
|----------|------|
| 아이디 입력 후 중복확인 클릭 | 아이디 중복 여부 확인 → 결과 표시 |
| 아이디 수정 시 | 중복확인 상태 초기화 → 다시 중복확인 버튼 클릭 필요 |
| 이메일 입력 후 인증번호 발송 클릭 | 이메일 중복 검사 → 중복 시 에러, 아니면 인증번호 발송 |
| 인증번호 입력 후 확인 클릭 | 인증번호 검증 → 성공/실패 표시 |
| 재발송 클릭 | 새 인증번호 발송 → 타이머 리셋 |
| 닉네임 입력 후 중복확인 클릭 | 닉네임 중복 여부 확인 → 결과 표시 |
| 닉네임 수정 시 | 중복확인 상태 초기화 → 다시 중복확인 버튼 클릭 필요 |
| 회원가입 버튼 클릭 | 전체 유효성 검사 (아이디 중복확인, 이메일 인증, 닉네임 중복확인 완료 포함) |
| 회원가입 성공 | 로그인 페이지로 이동 (`/login`) + 성공 메시지 |
| 회원가입 실패 | 에러 메시지 표시 |
| 로그인 링크 클릭 | 로그인 페이지로 이동 (`/login`) |

## 8. 에러 메시지

| 상황 | 에러 메시지 |
|------|-------------|
| 아이디 6자 미만 | 아이디는 6자 이상이어야 합니다 |
| 아이디 중복 | 이미 사용 중인 아이디입니다 |
| 아이디 중복확인 안함 | 아이디 중복확인을 해주세요 |
| 비밀번호 형식 불일치 | 비밀번호는 대문자, 소문자, 특수문자를 포함하여 12자 이상이어야 합니다 |
| 비밀번호 불일치 | 비밀번호가 일치하지 않습니다 |
| 이메일 형식 오류 | 올바른 이메일 형식을 입력해주세요 |
| 이메일 중복 | 이미 사용 중인 이메일입니다 |
| 이메일 인증 안함 | 이메일 인증을 완료해주세요 |
| 인증번호 불일치 | 인증번호가 일치하지 않습니다 |
| 인증번호 만료 | 인증번호가 만료되었습니다. 재발송해주세요 |
| 닉네임 2자 미만 | 닉네임은 2자 이상이어야 합니다 |
| 닉네임 중복 | 이미 사용 중인 닉네임입니다 |
| 닉네임 중복확인 안함 | 닉네임 중복확인을 해주세요 |
