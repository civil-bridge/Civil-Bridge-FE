# 경기파트너스 기능명세서

---

## 1. 로그인 화면

**화면 설명**: 가입한 계정으로 로그인하는 화면

**진입 경로**: 앱 첫 진입 시 (비로그인 상태)

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 로그인 아이디 입력창 | input | 아이디 입력 |
| 비밀번호 입력창 | input | 비밀번호 입력 (마스킹 처리) |
| 로그인 버튼 | button | 로그인 요청 |
| 회원가입 링크 | link | 회원가입 화면으로 이동 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 로그인 버튼 클릭 | 메인화면으로 이동 | 이메일/비밀번호 불일치 시 에러 메시지 표시 |
| 회원가입 링크 클릭 | 회원가입 화면으로 이동 | - |

---

## 2. 회원가입 화면

**화면 설명**: 새로운 계정을 생성하는 화면

**진입 경로**: 로그인 화면 → 회원가입 링크 클릭

**특이사항:** 아이디, 닉네임, 전화번호, 이메일 중복 불가능

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 아이디 입력창 | input | 로그인 아이디 입력 |
| 비밀번호 입력창 | input | 비밀번호 입력 |
| 비밀번호 확인 입력창 | input | 비밀번호 재입력 |
| 이름 입력창 | input | 사용자 실명 입력 |
| 닉네임 입력창 | input | 사용할 닉네임 입력 |
| 전화번호 입력창 | input | 사용자 전화번호 입력 |
| 이메일 입력창 | input | 이메일 입력 |
| 인증번호 발송버튼 | button | 입력한 이메일로 인증번호 발송 |
| 인증번호 입력창 | input | 6자리 인증번호 입력 |
| 인증번호 확인버튼 | button | 인증번호 검증 |
| 회원가입 버튼 | button | 회원가입 요청 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 인증번호 발송 버튼 클릭 | 이메일로 6자리 인증번호 발송 (유효시간 5분) | 이미 가입된 이메일이면 에러 메시지 |
| 인증번호 확인 버튼 클릭 | 인증 성공 시 다음 입력 필드 활성화 | 인증번호 불일치 또는 만료 시 에러 메시지 |
| 회원가입 버튼 클릭 | 가입 완료 후 로그인 화면으로 이동 | 아이디 중복, 비밀번호 불일치, 닉네임 중복, 전화번호 중복 등 에러 메시지 |

---

## 3. 메인화면 (전체 방 목록)

**화면 설명**: 로그인 후 기본 화면. 개설된 모든 방을 조회하는 화면

**진입 경로**: 로그인 성공 시

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 전체 방 조회 탭 | tab | 현재 탭 (활성 상태) |
| 내가 참여한 방 탭 | tab | 내가 참여한 방 목록으로 이동 |
| 내 정보 탭 | tab | 내 정보 화면으로 이동 |
| 방 목록 | list | 방 카드들이 나열됨 (제목, 설명, 지역, 접근권한 표시) |
| 페이지 번호 | pagination | 1, 2, 3... 형태로 페이지 이동 |
| 방 생성 버튼 | button | 방 생성 화면으로 이동 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 방 카드 클릭 | 해당 방 입장 | 접근 권한 없으면 에러 메시지 (공무원만 입장 가능한 방 등) |
| 페이지 번호 클릭 | 해당 페이지의 방 목록 표시 (15개씩) | - |
| 방 생성 버튼 클릭 | 방 생성 화면으로 이동 | - |

---

| 방 목록 | list | 내가 참여한 방 카드들이 나열됨 |
| 페이지 번호 | pagination | 1, 2, 3... 형태로 페이지 이동 |

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 방 카드 클릭 | 해당 방 입장 | - |
| 페이지 번호 클릭 | 해당 페이지의 방 목록 표시 (15개씩) | - |

## 7. 방 생성

**화면 설명**: 새로운 논의방을 생성하는 화면

**진입 경로**: 메인화면 → 방 생성 버튼 클릭

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 방 제목 입력창 | input | 방 제목 입력 |
| 방 설명 입력창 | textarea | 방에 대한 간단한 설명 입력 |
| 지역 선택 | dropdown | 시 단위 지역 선택 (부천시, 수원시 등) |
| 접근 권한 선택 | radio | 공무원만 / 모두 |
| 생성 버튼 | button | 방 생성 |
| 취소 버튼 | button | 이전 화면으로 돌아가기 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 생성 버튼 클릭 | 방 생성 후 해당 방에 자동 입장 | 필수 입력값 누락 시 에러 메시지 |
| 취소 버튼 클릭 | 메인화면으로 이동 | - |

---

## 8. 방 안 (채팅)

**화면 설명**: 논의방 내부. 참여자들과 실시간 채팅하는 화면

**진입 경로**: 방 목록에서 방 클릭하여 입장

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 방 정보 | text | 방 제목, 지역 등 표시 |
| 참여자 목록 | list | 현재 참여 중인 멤버 표시 (공무원은 뱃지로 구별) |
| 채팅 영역 | chat | 메시지 목록 (위로 스크롤 시 이전 메시지 로드) |
| 메시지 입력창 | input | 메시지 입력 |
| 전송 버튼 | button | 메시지 전송 |
| 메뉴 버튼 | button | 채팅방 옵션이 슬라이드로 나옴 |
| 새 제안서 작성 버튼 | button | 제안서 작성 화면으로 전환 |
| 제안서 불러오기 버튼 | button | 저장된 제안서 목록 표시 |
| 완료된 제안서 보기 버튼 | button | 동의 진행 중/완료된 제안서 목록 표시 |
| 방 나가기 버튼 | button | 방 퇴장 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 메시지 입력 후 전송 | 채팅 영역에 메시지 표시 (실시간) | - |
| 위로 스크롤 | 이전 채팅 기록 로드 | - |
| 새 제안서 작성 버튼 클릭 | 제안서 작성 화면으로 전환 | 이미 작성 중인 제안서 있으면 [미정] |
| 메뉴 버튼 클릭 | 제안서 및 나가기 관련 버튼이 나옴 |  |
| 제안서 불러오기 버튼 클릭 | 저장 상태 제안서 목록 표시, 선택 시 이어서 작성 | - |
| 완료된 제안서 보기 버튼 클릭 | 완료된 제안서 목록 표시 (읽기 전용) | - |
| 방 나가기 버튼 클릭 | 즉시 퇴장, 메인화면으로 이동 | 마지막 참여자면 방 삭제 |

---

## 9. 제안서 작성

**화면 설명**: 제안서를 작성하는 화면. 왼쪽 제안서 + 오른쪽 채팅으로 분할

**진입 경로**: 방 안 → 새 제안서 작성 버튼 클릭 또는 제안서 불러오기

---

**화면 요소:**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 제안서 영역 (왼쪽) | form | 제안서 작성 폼 |
| 제안서 제목 입력창 | input | 제안서 제목 |
| 제안서 내용 입력창 | textarea | 문제 개요 작성 |
| 해결방안 입력창 | textarea | 해결 방안 작성 |
| 첨부 자료 영역 | list | 증거자료 추가 (타입/제목/URL) |
| 첨부자료 추가 버튼 | button | 증거자료 항목 추가 |
| 채팅 영역 (오른쪽) | chat | 실시간 채팅 (기존과 동일) |
| 저장 버튼 | button | 현재 상태 저장 (화면 유지) |
| 나가기 버튼 | button | 제안서 작성 화면에서 나가기 |
| 제출 버튼 | button | 제안서 제출 (동의 투표 화면으로) |

---

**사용자 동작 (작성자):**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 제안서 내용 입력 | 다른 참여자들에게 실시간으로 보임 | - |
| 저장 버튼 클릭 | 현재 상태 저장, 화면 유지 | - |
| 나가기 버튼 클릭 | 저장 안 했으면 모달로 저장 유도, 그래도 나가면 내용 사라짐 | - |
| 제출 버튼 클릭 | 동의 투표 설정 화면으로 이동 | 필수 입력값 누락 시 에러 메시지 |

---

**사용자 동작 (나머지 참여자):**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 제안서 영역 | 실시간으로 보기만 가능 (수정 불가) | - |
| 채팅 입력 | 메시지 전송 가능 | - |

---

## 10. 동의 투표

**화면 설명**: 제안서 제출 시 동의 설정 및 투표하는 화면

**진입 경로**: 제안서 작성 → 제출 버튼 클릭

---

**화면 요소 (제출 시 설정 - 작성자):**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 최소 동의 인원수 입력 | input (number) | 제출 가능 조건 인원수 |
| 동의 마감기한 선택 | datetime picker | 마감 일시 선택 |
| 확인 버튼 | button | 동의 투표 시작 |

---

**화면 요소 (동의 진행 중 - 참여자):**

| 요소 | 타입 | 설명 |
| --- | --- | --- |
| 제안서 내용 | text | 완성된 제안서 (읽기 전용) |
| 현재 동의 현황 | text | N명 동의 / 최소 M명 필요 |
| 마감기한 | text | 마감까지 남은 시간 |
| 동의 버튼 | button | 동의 투표 |

---

**사용자 동작:**

| 동작 | 결과 | 예외 처리 |
| --- | --- | --- |
| 동의 버튼 클릭 | 동의 수 +1, 버튼 비활성화 | 이미 동의했으면 버튼 비활성화 상태 |
| 최소 인원 충족 | 제출 가능 상태, 민원사이트 링크 표시 | - |
| 마감기한 도달 | 동의 수 상관없이 제출 가능 상태, 민원사이트 링크 표시 | - |

---
