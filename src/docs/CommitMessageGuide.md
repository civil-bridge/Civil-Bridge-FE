# 1. 개요

---

> 커밋 메시지의 일관성을 위한 컨벤션 가이드.
> 

# 2. 메시지 형식

---

```
유형: 한줄 요약

- 변경사항1
- 변경사항2

```

- 한줄 요약은 간결하게 작성
- 바디는 선택사항. 부가 설명이 필요할 때만 작성
- 헤더와 바디 사이에 빈 줄 하나

# 3. 작업 유형

---

### 1. feat

---

<aside>
💡

새로운 기능 추가

</aside>

- 작성 예시:

```
feat: 회원가입 API 구현

- UserController 생성
- 이메일 중복 검증 로직 추가
```

### 2. fix

<aside>
💡

버그 및 오류 수정

</aside>

- 작성 예시:

```
fix: 로그인 시 세션 만료 오류 수정

- 토큰 갱신 로직 누락 수정
```

### 3. refactor

<aside>
💡

동작 변화 없이 코드 구조 개선

</aside>

- 패키지 구조 정리 포함
- 파일 이름 변경(의미 변경) 포함
- 작성 예시:

```
refactor: UserService 메서드 분리

- validateUser 메서드 추출
- 중복 코드 제거
```

### 4. style

<aside>
💡

코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음)

</aside>

- 파일 이름 오타 수정 포함
- 작성 예시:

```
style: 코드 포맷팅 적용
```

### 5. docs

<aside>
💡

README, 주석 등 문서 수정

</aside>

- 작성 예시:

```
docs: API 명세서 업데이트

- 회원가입 엔드포인트 추가
```

### 6. test

<aside>
💡

테스트 코드 추가 및 수정

</aside>

- 작성 예시:

```
test: UserService 단위 테스트 추가

- 회원가입 성공 케이스
- 이메일 중복 실패 케이스
```

### 7. chore

<aside>
💡

빌드 설정, 패키지 매니저, 기타 프로젝트 설정

</aside>

- 불필요한 파일/클래스 삭제 포함
- 작성 예시:

```
chore: Spring Security 의존성 추가
```

### 8. env환경변수, 설정 파일 변경

```
env: 운영 환경 DB 접속 정보 변경
```

### 9. perf

<aside>
💡

성능 향상을 위한 코드 변경

</aside>

- 작성 예시:

```
perf: 회원 목록 조회 쿼리 최적화

- N+1 문제 해결
- 페이징 적용
```

### 10. ci

<aside>
💡

CI 설정 파일 변경

</aside>

- 작성 예시:

```
ci: GitHub Actions 워크플로우 추가
```

### 11. revert

<aside>
💡

이전 커밋 되돌리기

</aside>

- 작성 예시:

```
revert: feat: 회원가입 API 구현

- 롤백 사유: 요구사항 변경
```
